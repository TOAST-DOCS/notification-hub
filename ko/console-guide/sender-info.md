<h1>발신 정보</h1>

**Notification > Notification Hub > 콘솔 사용 가이드 > 발신 정보**

메시지 채널별 발신 정보를 등록하고 관리합니다. 메시지를 발송하려면 각 메시지 채널에 맞는 발신 정보가 사전에 등록되어 있어야 합니다.

<span id="manage-sender-phone-number"></span>

## 발신 번호 관리

SMS, LMS, MMS 메시지를 발송하려면 발신 번호를 등록해야 합니다. 전기통신사업법 관련 고시에 따라 발신 번호 등록 시 명의자 인증이 필요합니다. 명의자 인증 시 회원 및 발신 번호 유형에 따라 인증 방법과 필요한 서류가 결정됩니다.

<span id="register-sender-phone-number"></span>

### 발신 번호 등록

1. **+ 발신 번호 등록**을 클릭하고 **개인정보 수집 이용 동의서**에 동의합니다.
2. 등록할 발신 번호 유형을 선택하고 발신 번호를 입력합니다.
3. 발신 번호 유형에 필요한 서류를 첨부합니다.
4. 등록 요청 후 심사가 승인되면 발신 번호가 등록됩니다.

발신 번호 사전 등록제는 **이용 정책 및 사전 설정 안내 > SMS > 발신 번호 사전 등록제 시행**을 참고하세요.

* [발신 번호 사전 등록제 시행 바로 가기](../service-policy-and-precondition/sms#sender-phone-number-pre-registration)

<span id="sender-phone-number-verification"></span>

### 명의자 인증 안내

| 회원 구분 | 발신 번호 유형 | 인증 방법 | 필요 서류 |
|---|---|---|---|
| 사업자 회원 | 대표자 번호, 사업자 자체 번호 | 서류 인증 | 통신서비스 이용증명원 |
| 사업자 회원 | 임직원 번호 | 서류 인증 | 통신서비스 이용증명원, 재직증명서 |
| 사업자 회원 | 타사 번호 | 서류 인증 | 통신서비스 이용증명원, 이용승낙서, (타사)사업등록증, 사업자와 타사 간 관계 확인 문서(계약서 등) |
| 사업자 회원 | 타인 번호 | 서류 인증 | 통신서비스 이용증명원, 이용승낙서 |

* 이용승낙서 기본 양식은 콘솔에서 내려받을 수 있습니다.
* 사업자와 타사 간 관계 확인 문서는 업무위수탁 계약서, 본점-지점 증빙 서류 등이 될 수 있습니다.
* 통신서비스 이용증명원은 **마스킹(숨김) 처리된 부분이 없고, 최근 3개월 이내 발급된 서류**만 가능합니다.
* 재직증명서는 **발급일이 표기되어 있고, 날인된 서류**만 가능합니다.
* 재직증명서 내 **주민등록번호 뒤 6자리는 반드시 마스킹(숨김) 처리**하세요. 예: 000000-0\*\*\*\*\*\*

<span id="sender-phone-number-format"></span>

### 발신 번호 입력 형식 안내

* 유선 전화번호: 02-YYY-YYYY(지역 번호 포함하여 등록)
* 이동통신 전화번호: 010-ABYY-YYYY
* 공통 서비스 식별번호: 0N0 계열(번호 앞에 지역 번호 사용 금지)
* 최소 8자리~최대 11자리까지의 발신 번호 입력 가능
* 존재하지 않는 번호 대역으로 메시지 전송 불가(예: 070-0YYY, 070-1YYY, 010-0YYY, 010-1YYY)

<span id="delete-sender-phone-number"></span>

### 발신 번호 삭제

등록된 발신 번호를 삭제할 수 있습니다.

1. 삭제할 발신 번호의 체크박스를 선택합니다.
2. **발신 번호 삭제**를 클릭합니다.

## 브랜드 관리

RCS 메시지를 발송하려면 RCS Biz Center에서 브랜드를 등록하고, NHN Cloud 콘솔과 연동해야 합니다.

<span id="brand-prerequisites"></span>

### 사전 준비

브랜드 연동 전 RCS Biz Center에서 다음 사항이 완료(승인)되어야 합니다.

1. RCS Biz Center 가입, 브랜드 생성 및 승인
2. 브랜드 운영 관리 메뉴에서 (주)NHN Cloud를 대행사로 지정
3. 대화방(발신 번호) 등록 및 승인

RCS Biz Center의 사전 등록 사항은 **이용 정책 및 사전 설정 안내 > RCS**를 참고하세요.

* [이용 정책 및 사전 설정 안내 > RCS 바로 가기](../service-policy-and-precondition/rcs)
* [RCS Biz Center 바로 가기](https://www.rcsbizcenter.com/main)

<span id="brand-sync"></span>

### 브랜드 연동

브랜드는 본인 인증 시 첨부한 사업자 등록증의 사업자 등록 번호를 기준으로 연동합니다.

* **+ 브랜드 연동**을 클릭하면 RCS Biz Center에 등록된 브랜드가 동기화됩니다.
* 브랜드 연동이 완료되면 브랜드 목록이 조회됩니다.
* RCS Biz Center에서 브랜드 정보를 변경한 경우 **브랜드 연동**을 클릭해 업데이트를 진행하세요.

## 도메인 관리

이메일을 발송하려면 자신이 소유한 도메인을 등록하고, 도메인 소유권을 인증해야 합니다. 도메인 인증 성공 후 SPF 레코드 인증, DMARC 인증, DKIM 사용 설정을 할 수 있습니다.

발신 도메인 및 SPF, DKIM, DMARC는 **이용 정책 및 사전 설정 안내 > 이메일**을 참고하세요.

* [이용 정책 및 사전 설정 안내 > 이메일 바로 가기](../service-policy-and-precondition/email)

<span id="register-domain"></span>

### 이메일 도메인 등록 및 소유권 인증

도메인을 등록하고 도메인 소유권을 인증해야 합니다. Notification Hub에서 제공한 값을 이메일 도메인 DNS TXT 레코드에 등록합니다. 제공된 값과 등록한 도메인의 TXT 레코드가 일치하는지 확인하여 소유권을 인증합니다.

1. **+ 도메인 등록**을 클릭합니다.
2. 등록할 루트 도메인을 입력하고 **검증**을 클릭합니다.
3. 검증에 성공하면 **등록**을 클릭해 등록을 마칩니다.
4. 도메인 목록에서 도메인 소유 인증 상태의 **인증**을 클릭합니다.

도메인 소유 인증에 성공하면 도메인 인증 상태가 **완료**로 변경됩니다.

<span id="spf-authentication"></span>

### SPF 인증

SPF(sender policy framework, 발신자 정책 프레임워크)는 이메일 발신자와 발송 서버의 신뢰성을 검증하기 위한 메커니즘으로, 이메일 수신 서버가 특정 도메인에서 발송된 메일이 실제로 허가된 이메일 발송 서버에서 왔는지 확인합니다.

SPF
```
v=spf1 include:_spfblocka.toast.com ~all
```

1. 위 SPF 항목의 SPF 레코드를 도메인 TXT 레코드에 등록합니다.
2. 목록에서 도메인을 선택합니다.
3. **SPF 레코드 인증 상태** 항목의 **상태 확인**을 클릭해 SPF 인증을 완료합니다.


!!! danger "주의 사항"
    * 도메인 TXT 레코드에는 하나의 SPF 레코드만 등록해야 합니다. 도메인 TXT 레코드에 2개 이상 SPF 레코드가 등록된 경우, SPF 인증이 실패해 이메일 수신 서버가 수신을 거부할 수 있습니다.
    * SPF 레코드를 검사할 때 DNS 조회를 발생시키는 메커니즘(include)과 수정자(redirect)의 사용은 최대 10개로 제한되며, 이를 초과하면 이메일 수신 서버에서 수신을 거부할 수 있습니다.

<span id="dkim-authentication"></span>

### DKIM 설정

DKIM(domainkeys identified mail, 도메인 키 식별 이메일)은 이메일 발송 서버가 이메일을 디지털 서명하고 이메일 수신 서버는 발신자 진위 여부를 확인하여 전송 중에 메시지가 위조, 변조되지 않았는지 확인하는 이메일 검증 방법입니다.

1. 도메인 소유 인증 완료 후 목록에서 도메인을 체크하고 **DKIM 설정**을 클릭합니다.
2. DKIM 인증을 위해 제공된 DNS 호스트 이름에 TXT 레코드 값을 설정하고 아래 **인증**을 클릭합니다.
    * 등록한 도메인이 `example.com`인 경우, `toast._domainkey.example.com` TXT 레코드에 값을 설정해야 합니다.
3. 인증 완료 후 사용 설정을 하고 **저장**을 클릭해 DKIM 인증을 완료합니다.

<span id="dmarc-authentication"></span>

### DMARC 인증

DMARC(domain-based message authentication reporting and conformance)는 이메일 보안 강화의 마지막 단계입니다. 이메일 스푸핑을 이용한 피싱, 사기 등을 막기 위한 도메인 기반 메시지 인증 보고 및 준수 정책입니다.

DMARC
```
v=DMARC1;p=none;sp=quarantine;pct=100;rua=mailto:${보고서를_수신할_이메일_주소}
```

1. 위 DMARC 항목의 값에 DMARC 보고서를 수신할 이메일 주소를 추가해 DMARC TXT 레코드를 완성합니다.
2. `_dmarc.`가 추가된 서브 도메인 TXT 레코드에 등록합니다.
    * 예: 도메인이 `example.com`인 경우 `_dmarc.example.com`의 TXT 레코드에 등록합니다.
3. **DMARC 인증 상태** 항목의 **상태 확인**을 클릭해 DMARC 인증을 완료합니다.

<span id="domain-protection"></span>

### 도메인 보호

도메인 보호가 활성화된 도메인은 다른 프로젝트에서 사용할 수 없습니다. 보호가 된 도메인을 다른 프로젝트에서 사용하기 위해서는 동일하게 도메인 등록과 소유 인증을 받아야 합니다.

!!! danger "주의 사항"
    도메인 보호를 비활성화하면 다른 프로젝트에서 임의로 도메인을 사용할 수 있습니다. 모든 인증을 완료한 도메인의 경우 다른 프로젝트에서 발송하는 이메일도 동일하게 이메일 수신 서버에서 정상적으로 수신됩니다. 이렇게 발송된 이메일이 스팸이거나 피싱인 경우 수신자에게 피해가 발생할 수 있고 도메인의 평판이 하락해 수신 이메일 서버에서 수신을 거부할 수 있습니다.

<span id="delete-domain"></span>

### 도메인 삭제

등록된 도메인을 삭제할 수 있습니다.

1. 삭제할 도메인의 체크박스를 선택합니다.
2. **도메인 삭제**를 클릭합니다.

## 푸시 인증 관리

Push 메시지를 발송하려면 푸시 서비스에서 발급하는 인증 정보를 등록해야 합니다.

Push 인증 정보 발급 방법은 **이용 정책 및 사전 설정 안내 > Push**를 참고하세요.

* [이용 정책 및 사전 설정 안내 > Push 바로 가기](../service-policy-and-precondition/push)

<span id="fcm-authentication"></span>

### FCM 인증 설정

Android 기기에 푸시 메시지를 발송하려면 FCM(Firebase Cloud Messaging) 인증 설정이 필요합니다.

1. **FCM 서비스 계정 키 등록**의 **설정 변경**을 클릭합니다.
2. 서비스 계정 키(JSON)에 발급받은 FCM Service Account Credential 파일의 내용을 복사해 붙여 넣습니다.
3. **검증 > 저장**을 클릭해 설정을 완료합니다.

<span id="apns-authentication"></span>

### APNS 인증 설정

iPhone에 푸시 메시지를 발송하려면 APNS(Apple Push Notification Service) 인증 설정이 필요합니다.

1. **APNS JWT 인증서 등록**의 **설정 변경**을 클릭합니다.
2. **팀 ID**와 **키 ID**를 입력합니다.
3. **토픽**을 입력합니다. 토픽은 앱의 Bundle ID입니다.
4. **개인 키** 파일의 내용을 복사해 붙여 넣습니다.
5. **검증 > 저장**을 클릭해 설정을 완료합니다.

<span id="adm-authentication"></span>

### ADM 인증 설정

Amazon Kindle, Fire 등의 기기에 푸시 메시지를 발송하려면 ADM(Amazon Device Messaging) 인증 설정이 필요합니다.

1. **ADM 자격 증명 등록**의 **설정 변경**을 클릭합니다.
2. **클라이언트 ID**와 **클라이언트 키**를 입력합니다.
3. **검증 > 저장**을 클릭해 설정을 완료합니다.

## 발신 프로필 관리

알림톡, 브랜드 메시지를 발송하려면 카카오톡 발신 프로필을 등록해야 합니다.

<span id="sender-profile-prerequisites"></span>

### 사전 준비

발신 프로필을 등록하려면 카카오톡 채널이 생성되어야 합니다. 카카오톡 홈페이지에서 카카오톡 채널을 생성하세요.

* [카카오톡 채널 생성 바로 가기](https://center-pf.kakao.com/)

발신 프로필을 등록하려면 카카오톡 채널 등록 후 비즈니스 인증을 받아야 합니다.

발신 프로필 생성 방법은 **이용 정책 및 사전 설정 안내 > 알림톡/브랜드 메시지**를 참고하세요.

* [이용 정책 및 사전 설정 안내 > 알림톡/브랜드 메시지 바로 가기](../service-policy-and-precondition/alimtalk-and-friendtalk)

<span id="register-sender-profile"></span>

### 발신 프로필 등록

1. **+ 발신 프로필 등록**을 클릭합니다.
2. 발신 프로필 ID, 관리자 휴대폰 번호, 카테고리를 설정한 뒤 **토큰 요청**을 클릭합니다.
3. 관리자 휴대폰으로 전송된 토큰 번호 6자리를 입력합니다.
4. **확인 > 등록**을 클릭하면 발신 프로필 등록이 완료됩니다.

!!! danger "주의 사항"
    알림톡 일별 **최대 발송량 기본값은 1,000건**입니다. 일별 최대 발송량을 변경하려면 [고객 센터](https://www.nhncloud.com/kr/support/inquiry)로 문의하세요.

* **발신 프로필 그룹 관리**를 클릭해 그룹을 생성하고 발신 프로필을 그룹에 추가할 수 있습니다.

## 080 수신 거부 번호 관리

080 수신 거부 번호는 광고 문자 발송 시, 수신자에게 수신 거부를 제공하는 서비스입니다. 광고성 메시지 발송 시 수신자가 수신 거부나 수신 동의의 철회를 무료로 할 수 있도록 **무료 수신 거부 방법을 반드시 기재**해야 합니다.

!!! danger "주의 사항"
    이를 어기는 경우 '정보통신망 이용촉진 및 정보보호 등에 관한 법률'에 의거하여 **3,000만 원 이하의 과태료**가 부과될 수 있습니다.

<span id="register-080-number"></span>

### 080 수신 거부 번호 신청

1. **+ 080 수신 거부 번호 신청**을 클릭합니다.
2. 회사명을 입력합니다. 입력한 회사명은 080 수신 거부 번호로 전화를 걸 때 안내되는 업체명입니다.
3. 신청을 완료하면 심사 중 상태로 변경됩니다.
4. 080 수신 거부 서비스 개통은 영업일 기준 3~4일이 소요되고, 개통이 완료되면 사용할 수 있습니다.

<span id="cancel-080-number"></span>

### 080 수신 거부 번호 이용 해지

* 등록된 080 수신 거부 번호의 체크박스를 선택하고 **080 수신 거부 번호 이용 해지**를 클릭합니다.
* **사용 중 - 외부 등록**, **사용 중 - 공유 번호** 상태인 080 번호는 이용 해지가 불가합니다.
